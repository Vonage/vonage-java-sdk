22     * @param from the 'from' address that will be seen on the handset when this message arrives,
23     *             typically either a valid short-code / long code that can be replied to, or a short text description of the application sending the message (Max 11 chars)
24     * @param to   the phone number of the handset that you wish to send the message to
25     * @param binaryMessageBody The raw binary message data to be sent to a handset.
26     *                          This api, and the Nexmo messaging service will send this data 'as-is' (in conjunction with the binary UDH) and will not make any corrections.
27     *                          so you should ensure that it is a correctly constructed message

31     *                         using the automated long messaging handling in the Nexmo messaging service, then you will need to construct and include here an appropriate
32     *                         UserDataHeader field that describes the segmentation/re-assembly fields required to sucessfully concatenate multiple short messages.
33     */
